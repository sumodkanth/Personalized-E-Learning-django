PHP preg_match() function

The preg_match() function is a built-in function of PHP that performs a regular expression match. This function searches the string for pattern, and returns true if the pattern exists otherwise returns false.

PHP - The __construct Function

A constructor allows you to initialize an object's properties upon creation of the object.
If you create a __construct() function, PHP will automatically call this function when you create an object from a class.
Notice that the construct function starts with two underscores (__)!

PHP array_filter() Function

This built-in function in PHP is used to filter the elements of an array using a user-defined function which is also called a callback function. The array_filter() function iterates over each value in the array, passing them to the user-defined function or the callback function. If the callback function returns true then the current value of the array is returned into the result array otherwise not. This way the keys of the array gets preserved, i.e. the key of element in the original array and output array are same.

What is set_exception_handler?

set_exception_handler is a built-in PHP function that allows you to set a global exception handler for your PHP application. This means that any uncaught exceptions in your application will be passed to this function for handling. By setting a custom exception handler, you can ensure that your application handles errors consistently and effectively, without relying on default error messages or behaviors.

PHP array_reverse() Function

This inbuilt function of PHP is used to reverse the elements of an array including the nested arrays. Also, we have an option of preserving the key elements according to the user’s choice. This function accepts an array as a parameter and returns the array with elements in reversed order.

Exception Handling in PHP

In PHP, exception handling is a mechanism that allows you to gracefully handle errors and exceptions that may occur during the execution of your code. The try, catch, and finally blocks are essential components of PHP's exception handling system.

1.try block:

  • The try block encloses the code that might throw an exception. It is the section of code where you anticipate that an exception might occur.
  • f an exception is thrown within the try block, the normal flow of execution is interrupted, and PHP immediately jumps to the appropriate catch block.

2.catch block:

  • The catch block is responsible for catching and handling exceptions thrown within the corresponding try block.
  • It specifies the type of exception to catch (in the example above, Exception is a generic class that can catch any exception), and it contains the code that should be executed when that specific exception is caught.
  • Multiple catch blocks can be used to handle different types of exceptions.

3.finally block:

  • The finally block, if present, contains code that will be executed regardless of whether an exception is thrown or not. It is useful for cleanup operations, such as closing files or database connections.
  • The code in the finally block is guaranteed to be executed, even if an exception occurs and is caught.

Autoloading classes in PHP

In order to use a class defined in another PHP script, we can incorporate it with include or require statements. However, PHP’s autoloading feature does not need such explicit inclusion. Instead, when a class is used (for declaring its object etc.) PHP parser loads it automatically, if it is registered with spl_autoload_register() function. Any number of classes can thus be registered. This way PHP parser gets a last chance to load the class/interface before emitting an error.
